Conclusions October 1, 2003 

  • Some thoughts about Bioperl:
    • Bioperl is a somewhat large collection of classes
    • The quality, consistency, and documentation can vary substantially
    • IMHO Bioperl gives the most predictable results for the simplest tasks (and data)
    • As with Chado and other "semi-structured" data models, TMTOWTDI

  • Scoping the task/defining requirements:
    • The scope of this task will likely depend quite heavily on the requirements
    • What could/would we use Bioperl for?
    • What would other people do with a more capable Bioperl BSML adaptor?
    • Are there any specific applications that need to be supported?
      • Knowing this would help in terms of scalability, performance requirements.
