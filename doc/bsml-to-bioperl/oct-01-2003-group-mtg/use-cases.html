Potential use-cases for a new Bioperl BSML adaptor October 1, 2003 

  • Leveraging Bioperl-compatible tools and software:
    • Apollo (e.g., via Bioperl support for GAME XML)
    • GBrowse (e.g., via Bioperl support for GFF flat file databases)
    • DAS server for Chado/BSML data?

  • Some of the tools and file formats supported in Bioperl:
    • Sequence file formats:
      • abi, ace, alf, chadoitext, chadosxpr, chadoxml (*), ctf, EMBL,
        FASTA, GAME XML, GCG, GFF, GenBank, PIR, SWISS-PROT
    • Search and alignment tools:
      • BLAST, Clustalw, EMBOSS (needle, water), exonerate, FASTA,
        HMMER, mase, MEGA, MEME, MSF, NEXUS, Pfam, Phylip,
        Prodom, PSI-BLAST, Selex, WABA
    • Web-accessible database servers:
      • AceDB, BioFetch, dbfetch, EMBL, GDB, GenBank, GenPept,
        RefSeq, SwissProt, XEMBL

  • Facilitating data transfers between TIGR and clients/collaborators:
    • Allow loading of Bioperl-supported formats into Chado (and vice-versa):
      • Format X -> Bioperl -> TIGR-BSML -> BSML Perl object layer -> Chado (*)
    • Data deliverables could be packaged with a standard API in addition to a MySQL DB
